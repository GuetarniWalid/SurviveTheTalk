## Challenges

### 1. Qwen Thinking Mode (Critical Discovery)
- **Impact:** 16s TTFT — would have been a kill gate FAIL
- **Root cause:** OpenRouter/Qwen3.5 Flash activates thinking mode by default, poorly documented
- **Resolution:** `reasoning.enabled=false` in extra_body + `/no_think` prompt prefix → 1.1s TTFT
- **Assessment:** Normal discovery-phase friction. Fast pivot, satisfying result

### 2. Pipecat API Deprecations (3/4 stories)
- `allow_interruptions` deprecated → `MinWordsUserTurnStartStrategy`
- `roomOptions` moved from `Room.connect()` → `Room()` constructor
- `on_participant_left` takes 3 args, not 2
- Import paths: `openrouter` vs `openrouter.llm`
- **Assessment:** Young framework (v0.0.108), fast-moving API. Documented in dev notes for future reference

### 3. Infrastructure Surprises (2/4 stories)
- VPS firewall (UFW) blocks port 8000 — must use port 80 via Caddy
- Hetzner renamed CX22 → CPX22
- Ubuntu 24.04 uses `ssh` service name, not `sshd`
- `.python-version` defaulted to 3.13 by `uv init`
- **Assessment:** Minor friction, quickly resolved

### 4. Sprint-Status Not Updated (Critical Process Gap)
- Dev forgot to update `sprint-status.yaml` from `ready-for-dev` → `in-progress` → `review`
- Reviewer forgot to update from `review` → `done`
- Walid had to manually update multiple times, causing team confusion
- Story files show `review` status while sprint-status shows `done` — inconsistency
- **Assessment:** Systemic issue requiring process change (see Action Items)

## Key Insights

1. **Detailed story specs are the #1 velocity multiplier.** Exact imports, verified patterns, "What NOT to Do" → single-pass execution
2. **Sprint-status must be treated like pre-commit checks.** Non-negotiable update at every transition
3. **Discovery-phase API friction is normal.** Document findings to build institutional knowledge
4. **Technical brief before creative work is non-negotiable.** Rive 0.14.x constraints must be communicated before .riv creation
5. **PoC-first approach validated.** Zero wasted effort on screens/features before core pipeline was proven

---

## Known Issues Carried Forward (from poc-validation-report.md)

1. **Silence handling:** Pipeline doesn't auto-respond to extended silence (24s tested)
2. **Cold start 3-8s:** Subprocess spawn + LiveKit handshake. Fix: worker pool (MVP scope)
3. **VAD stop_secs mismatch:** 0.3s vs recommended 0.2s — potential 100ms latency improvement
4. **Response length:** Marcus occasionally generates 3 long sentences — affects latency and TTS cost
5. **Barge-in sensitivity:** min_words=3 blocks quick yes/no during bot speech
6. **TTS cost at scale:** $0.053/call (90% of total) — Cartesia Enterprise pricing needed
7. **Break-even higher than PRD:** ~80 subscribers (not 60) due to Cartesia Scale plan fixed cost
8. **Double user messages:** Pipeline sometimes sends two consecutive user messages on mid-sentence pauses
