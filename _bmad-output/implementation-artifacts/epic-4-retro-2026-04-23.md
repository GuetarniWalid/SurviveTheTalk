## Successes & Strengths

### 1. Figma-extract → story-spec → implementation workflow (win of the epic)

Walid extracts Figma designs to local files and references them by relative path inside story specs. This avoided hand-specifying every pixel detail, kept story specs focused on intent, and enabled late-stage design iteration at low cost (see 4.5 double iteration: Rive puppet unification + Figma redesign post-implementation).

**Why it worked:** token-enforced design system (4.1b) absorbs iteration without visual drift elsewhere. Without `AppColors` + the hex-literal enforcement test, late iteration would have been painful.

### 2. "Validate fast, iterate on render" strategy assumed as MVP choice

The 1-screen → 2-screen split on Story 4.4 is not a process defect — it is the expected cost of Walid's deliberate strategy: validate specs quickly to unblock dev, judge on the rendered output, iterate. For an MVP, this is the correct trade-off. The alternative (freezing every detail upfront) would cost far more decisions made blind.

### 3. "Deliberate Deviations" section + Claude Opus 4.7

The formalized "Deliberate Deviations" section (emerged in 4.4, carried into 4.5) documents the dev's intentional spec divergences so the reviewer doesn't re-flag them. Combined with Opus 4.7, this raised signal-to-noise in reviews dramatically: 4.5 had fewer patches than 4.2 despite being the most-deviated story of the epic.

### 4. "Previous Story Intelligence" propagation

Hard-won Flutter knowledge (setMockInitialValues, sealed-class mock fallback, `pumpAndSettle` hangs on continuous animations, sync-redirect pattern) was actively transmitted in the "Previous Story Intelligence" sections of 4.4 and 4.5 — visibly thicker than in 4.1/4.1b.

### 5. Reusable patterns established (epic-wide breakthroughs)

- `bootstrap()` + `FlutterError.reportError` init-failure pattern (4.1)
- Token-enforcement test scanning `lib/` for hex literals (4.1b) — forcing function against visual drift
- `AUTH_DEPENDENCY` module-level export (4.2) — one-liner protected routers
- `ok()`/`err()`/`now_iso()` envelope helpers (4.2)
- Atomic CAS claim replacing check-then-mutate (4.2)
- `refreshListenable` + `GoRouterRefreshStream` + sync cache (4.3 → 4.4 → 4.5)
- `preload()` + `hasXSync` storage cache pattern (4.4)
- Inline error Text as the epic-wide error-display convention (4.3)
- Thin service-wrapper pattern (PermissionService 4.4, VibrationService 4.5)
- `CharacterAvatar` Rive-enum widget (4.5) — carries into Epic 6
- Error UX = bounce to safe fallback, not inline retry (4.5)

### 6. Adversarial reviews caught real issues before any user

- 4.1b: 2 real WCAG AA failures (onError/onSecondary contrast) caught at foundation layer, corrected before any screen consumed the broken tokens.
- 4.2: TOCTOU race in verify-code (closed via atomic CAS); `user_id: true` forgery gap (bool-is-int subclass); empty JWT secret accepted; missing `exp` claim accepted; multi-worker migration race.
- 4.5: Popen ordered before DB insert (no rollback), double-submit on Accept button, emit-after-close bloc race.

## Challenges & Growth Areas

### 1. Repeated Flutter gotchas across stories

Trap knowledge was propagated via "Previous Story Intelligence" but still rediscovered multiple times because there was no canonical home:

- `FlutterSecureStorage.setMockInitialValues({})` missing in setUp → test crashes (4.3, 4.4, 4.5)
- `pumpAndSettle` hangs on continuous animations (CircularProgressIndicator, RingAnimation, Timer-driven text) → explicit `pump(Duration(ms))` required
- Sealed-class mock fallback: `Fake extends` blocked → `registerFallbackValue(ConcreteEvent())`
- `prefer_const_constructors` lint surging on every story
- `verifyNever(...)` not `verify(...).called(0)` on mocktail
- Test viewport too large to catch overflow → `setSurfaceSize` + `addTearDown`

### 2. Reactivity bugs (Bloc / GoRouter)

- **Same-const state skip:** emitting the same `const SomeState()` twice → BlocListener silently drops the second fire. Caused the "infinite loader" bug on mic permission recheck (4.4).
- **Async GoRouter redirect → flash-of-wrong-content** during FlutterSecureStorage I/O. Forced emergency adoption of the `preload() + hasXSync` pattern (4.4).

### 3. Recurring reviewer themes

- Silent error swallowing (4.1 bootstrap catch blocks, 4.5 `/calls/initiate`)
- Side-effect atomicity/ordering (4.2 TOCTOU → atomic CAS; 4.5 Popen must be LAST with DB rollback on spawn failure)
- Type-coercion security gaps (4.2: bool-is-int `user_id` forgery; empty JWT secret; missing exp claim)
- Multi-worker / concurrency races (4.2 migration race, invalidate_previous_codes race)

### 4. Post-implementation UX iteration happened twice on the epic

4.3 (toast + warning color added post-dev) and 4.5 (Rive unification + Figma redesign, two separate iterations). This is **not framed as a problem** — it is the expected surface area of Walid's validate-fast-then-iterate strategy, and the Figma-extract workflow absorbs the cost. Captured here for transparency.
